## 🌳 브랜치 전략 및 디렉토리 구조

### ♟️ 브랜치 전략

- README.md 파일의 충돌을 박기 위해 `Pull Request`하여 `main`브랜치에 `merge`한다.
- 브랜치명은 문제명을 그대로 사용한다.
- 해당 문제에 대한 회고까지 완료한 뒤에 `PR`을 한다.
- `merge` 후 로컬에서는 main branch에서 `pull`을 한다.
- `pr`후 문제 `branch`는 삭제한다.

#### 🔖 PR 컨벤션

`[{카테고리 또는 문제 번호}_{문제 제목}] 예시 문제`

- e.g.: [프로그래머스] 예시 문제
- e.g.: [BOJ/0000] 예시 문제
- e.g.: [디렉토리정리]
